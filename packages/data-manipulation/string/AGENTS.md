# Agent Instructions

This file provides guidance to AI coding agents when working with code in this directory.

## Overview

`@visulima/string` is the broad string-manipulation toolbox: case conversion (camel/pascal/snake/kebab/constant/dot/path/sentence/title/train/flip/flat/no), slug + transliteration, width-aware slice/truncate/word-wrap/align, indent/dedent/outdent, Levenshtein-based similarity, east-asian width detection, emoji-safe string handling, and a Vitest helper module.

## Architecture

- **Many sub-exports** — the `package.json` `exports` field maps ~30 deep paths (`./case/camel-case`, `./align-text`, `./slugify`, `./truncate`, `./get-string-width`, `./outdent`, `./test/vitest`, etc.). Most map 1:1 to a `src/<file>.ts`; case helpers live under `src/case/`. When adding a new function, add both the source file and an `exports` entry.
- **Generated code**: `src/charmap/` contains ~150 `block-*.ts` files generated by `scripts/generate-charmap.mjs` from `data/transliteration.json`. **Do not edit** files in `src/charmap/` by hand — they are regenerated by `pnpm run generate:charmap`, which is wired into `build`, `build:prod`, and all `test*` scripts. If the build looks stale, run it explicitly.
- **Non-default scripts** to be aware of: `generate:charmap` runs ahead of every build/test; `test` also passes `--typecheck` to Vitest (this package validates type-level helpers in `native-string-types.d.ts` and `src/case/types.ts`).
- `./test/utils` and `./test/vitest` sub-exports are intentional test-helper modules consumed by other packages — keep them ship-ready.
- Type-level case utilities (`CamelCase`, `PascalCase`, ...) are exported from `src/case/types.ts`; they mirror the runtime functions and should stay in sync.
- Tiny indirect deps (`indent-string`, `redent`, `strip-indent`, `emoji-regex-xs`, `fastest-levenshtein`, `get-east-asian-width`) are devDependencies inlined by `packem`.

## Related

- `@visulima/colorize` is pinned as a devDependency for ANSI-aware width tests.
- Width helpers (`getStringWidth`, `getStringTruncatedWidth`) are heavily used by terminal packages (`@visulima/pail`, `@visulima/boxen`).
