# vis flaky

Aggregates task outcomes from past run summaries (`.task-runner/runs/*.json`) and displays a flakiness report. A task is considered flaky when it has both successes and failures across multiple runs.

## Prerequisites

Run summaries must exist — pass `--summarize` to `vis run`:

```bash
vis run build --summarize
```

Each run writes a JSON summary to `.task-runner/runs/`. The `vis flaky` command reads these files.

## Usage

```bash
vis flaky [options]
```

## Examples

```bash
vis flaky                     # Show all flaky tasks (min 2 runs)
vis flaky --min-runs=5        # Require at least 5 runs
vis flaky --since=2026-01-01  # Only recent runs
vis flaky --json              # Machine-readable output
```

## Output

```
Task              Runs  Failures  Rate   Last Failure
────────────────  ────  ────────  ─────  ─────────────────────
@myorg/api:test   12    3         25.0%  2026-04-10T14:23:00Z
@myorg/web:build  8     1         12.5%  2026-04-08T09:15:00Z

2 flaky task(s) detected across 20 run(s).
Tip: use retryCount in project.json options to auto-retry flaky tasks.
```

## Options

| Option       | Default | Description                                        |
| ------------ | ------- | -------------------------------------------------- |
| `--min-runs` | `2`     | Minimum recorded runs to consider a task           |
| `--since`    |         | Only runs after this ISO 8601 date                 |
| `--json`     | `false` | Emit JSON instead of a table                       |

## Acting on Flaky Tasks

Once you've identified flaky tasks, add `retryCount` to their target options:

```json
{
    "targets": {
        "test": {
            "command": "vitest run",
            "options": {
                "retryCount": 2,
                "retryDelay": "exponential"
            }
        }
    }
}
```
