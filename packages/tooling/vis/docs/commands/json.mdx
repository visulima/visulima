# vis json

Power-user surface for piping workspace data into `jq`, `awk`, or other line-oriented tools. Each subcommand defaults to NDJSON (one record per line) so output streams cleanly through `| sort | uniq -c` and friends.

## Subcommands

### `vis json deps`

Emit every dep-instance across the workspace. One record per `(declaring package, dep block, dep name)` triple — counts the same dep N times if N packages declare it.

```bash
vis json deps                                          # NDJSON, every instance
vis json deps --internal-only                          # Only workspace/internal deps
vis json deps --external-only --dep-type=dependencies  # External runtime deps
vis json deps --include '@scope/*'                     # Filter declaring packages
vis json deps --exclude '@scope/internal-*'            # Drop matching declaring packages
vis json deps --format=json --pretty                   # Single pretty-printed array
vis json deps | jq -r '.depName' | sort -u             # Unique deps across the workspace
```

#### Record shape

Each NDJSON line is one of:

```json
{
    "packageName": "@my/app",
    "packageJsonPath": "apps/app/package.json",
    "depType": "dependencies",
    "depName": "react",
    "specifier": "^18.2.0",
    "internal": false
}
```

`packageJsonPath` is always workspace-relative for portability across CI runners. `internal: true` indicates the dep name resolves to another workspace package.

`pnpm-workspace.yaml#overrides` (pnpm v9+ moved `pnpm.overrides` from `package.json` into the workspace config) is read automatically and surfaced under `depType: "pnpm.overrides"`. The `packageJsonPath` for those entries points at `pnpm-workspace.yaml` so downstream tools know where to write back.

#### Options

| Option            | Default  | Description                                                                                                                                |
| ----------------- | -------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| `--include`       |          | Glob of declaring package names to keep. Repeatable.                                                                                       |
| `--exclude`       |          | Glob of declaring package names to drop. Repeatable.                                                                                       |
| `--dep-type`      |          | Restrict to a specific dep block (`dependencies`, `devDependencies`, `peerDependencies`, `optionalDependencies`, `overrides`, `resolutions`, `pnpm.overrides`). Repeatable. |
| `--internal-only` | `false`  | Only emit internal/workspace deps.                                                                                                         |
| `--external-only` | `false`  | Only emit external/registry deps.                                                                                                          |
| `--format`        | `ndjson` | Output format: `ndjson` (default, line-streamed) or `json` (single array).                                                                 |
| `--pretty`        | `false`  | Pretty-print with 2-space indent. Only meaningful with `--format=json`.                                                                    |

#### Recipes

Count how many packages declare each dep:

```bash
vis json deps | jq -r '.depName' | sort | uniq -c | sort -rn | head
```

List every package that pins a specific dep:

```bash
vis json deps | jq -r 'select(.depName == "react") | "\(.packageName)\t\(.specifier)"'
```

Find specifier drift for a single dep (alternative to `vis lint --workspace-versions --dep <name>`):

```bash
vis json deps | jq -r 'select(.depName == "react") | .specifier' | sort -u
```

Audit which packages still declare a banned dep before turning on `vis lint --banned-deps`:

```bash
vis json deps | jq -r 'select(.depName | IN("left-pad","request","moment")) | "\(.packageName)\t\(.depName)"'
```
